经典场景，我们通过网络活动IOC、落地时间等信息定位到恶意进程以后，再通过进程ID（PID）来定位到样本。下一步就是要通过样本的路径来定位到启动项。完整理解恶意进程的活动流程，彻底查杀恶意程序。

# Linux


| 方式 | 适用场景 | 典型路径 |
| ------ |------ |------ |
| **Systemd** | 后台服务、服务器软件、数据库 | /etc/systemd/system/ |
| **rc.local** | 简单的临时脚本、兼容老系统 |  /etc/rc.local |
| **Crontab** | 周期性任务、一次性初始化 | /var/spool/cron/ |
| **Autostart** | 桌面图形界面软件 |  ~/.config/autostart/ |
| **Init.d** | 老旧系统、传统脚本管理 |  /etc/init.d/ |

能够持续把恶意程序拉起的地方非常多，不只上述这些地方，也可能藏到任何一个正常启动项中的脚本中，手工几乎无法排查。但是可以注意到，最核心的位置基本上都在/var /etc /usr /opt 等目录下。因此我们准备一个常用脚本，使用grep命令来搜索所有这些目录下的所有文件，查找是否有可疑的样本路径。
例如我们已经定位到恶意程序位置：/usr/local/bin/malicious

    ```bash
    # 搜索所有目录下的所有文件，查找是否有可疑的字符串
    grep -rnI --exclude-dir="/var/log" "/usr/local/bin/malicious" /etc /var /opt /usr
    ```
-I: 忽略二进制文件。由于系统的启动项配置（如 Systemd 服务、计划任务、脚本等）全都是纯文本文件，加上这个参数后，grep 遇到二进制程序或库文件时会直接跳过，从而极大地提升搜索速度。
-r: 递归搜索。
-n: 显示匹配的行号。
--exclude-dir: 排除指定目录。这里排除了/var/log目录，因为日志文件通常非常多，搜索这些文件会非常耗时。



# Windows

Windows 下的持久化机制极其繁杂，手工逐一排查效率极低。

微软 Sysinternals 套件中的 Autoruns 是查看系统自启目录的首选工具。包含常规注册表项（Logon）、计划任务（Scheduled Tasks）、系统服务（Services）以及 WMI 等高级隐藏启动项。重点关注标红或高亮的项



另外，提供一个bat脚本，可以方便导出所有计划任务到schtasks.csv文件中。

```cmd
@echo off

%1 mshta vbscript:CreateObject("Shell.Application").ShellExecute("cmd.exe","/c %~s0 ::","","runas",1)(window.close)&&exit

cd/d "%~dp0"

SCHTASKS /Query /FO csv /V /NH > ./schtasks.csv
```
